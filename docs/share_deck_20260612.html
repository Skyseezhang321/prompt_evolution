看不懂表没关系，记三条暗线：
  1. 反馈信息密度决定样本效率——GEPA 用 ~3,936 rollouts 超过用 24,000 的 GRPO（RL），省 35 倍。
  2. 优化对象升维靠程序抽象——单指令 → 指令+示例 → 多模块系统。
  3. 搜索结构是独立维度——盲采样 → beam → MCTS → Pareto 池；PromptAgent 同等探索量消融（MCTS 0.754 vs beam 0.697）与 GEPA「Pareto 选择是主要贡献」互证。

  2. 跨模型极脆弱：同一套配置换到 Nova Lite，6/6 全胜崩到 1/6。「在模型 X 上有效」不可默认迁移。

纪律：任何复杂方法，对照组至少放 APE 和 OPRO 两个下限基线，打不过就别上。跨论文的原始分数永远不可横比。

06 · 方法地图选型六问：问完再掏钱

  1. 有留出集和评分器吗？没有 → 先别选方法，去凑五件套。
  2. headroom 体检过了吗？提升空间没超过噪声地板 → 停。
  3. 有 dev set？先跑示例优化基线——多数任务它比改指令更有效。
  4. 任务有「少量样例表达不了的隐藏规则」？有 → 才值得优化 instruction。
  5. 系统能产出可解释 trace？能 → 反思式方法（GEPA 类）才有信息优势；只有标量分 → 用 APE/OPRO 风格。
  6. 打得过下限基线吗？打不过 APE/OPRO 的复杂方法不值得引入。

结构记法：前两问定「值不值得做」 · 中间三问定「该改哪、用什么信号」 · 最后一问定「值不值这个复杂度」——多数团队的问题出在前两问根本没问。

07 · 精选洞见 1/5洞见 01 · 动手之前先体检 A

量两个数，再决定要不要花钱：

  • headroom：最好的候选比 zero-shot 高多少？
  • noise floor：候选之间本来就抖多少？
  • headroom 没超过 noise floor → 停，别优化。

典型翻车：30 条小样本上 dev 82→87「涨了」，100 条新数据 79——那 5 个点是噪声里捡的。

49%Coin Flip：72 次优化低于 zero-shot 的比例
~3 步ProTeGi：约 3 步达峰，之后在训练集上过拟合

08 · 精选洞见 2/5洞见 07 · 先改示例，不是改指令 A

大家的本能是打磨 instruction 措辞。但有 dev set 时，换「给模型看哪几个例子」常常更有效——专挑覆盖易错边界的 3 个。

分诊规则（MIPROv2）：多数任务 demo 优化 > instruction 优化；只有当任务含「隐藏条件规则、少量样例表达不了」时，改指令才是决定性的。

09 · 精选洞见 3/5洞见 13 · 今晚就能试：把 prompt 原样重复一遍 Arecent

QUERY → QUERY + QUERY

机制：causal attention 是单向的——prompt 前段的 token「看不见」后段。原样重复后，第二遍的每个 token 都能回头看见完整第一遍。

  • 不改语义、不增加输出长度和时延（重复发生在可并行的 prefill）
  • Padding 对照无效→ 收益来自重复本身，不是变长
  • 边界：推理模式下基本无效（28 组仅 5 胜）；输入 token 计费 ×2
47/70 胜 0 负非推理模式，7 个主流模型 × 10 配置（McNemar p<0.1）
21.33% → 97.33%NameIndex 长上下文索引任务（Gemini 2.0 Flash-Lite）

用法：把它设成所有优化实验的及格线——任何花钱搜出来的提升，先回答「打得过这个零智能变换吗」。

10 · 精选洞见 4/5洞见 06 · 越改越长，是病不是进步 A

优化后更长、规则更多、看起来更「专业」——多半是在给训练样本打补丁：

- 提到「发票」且含数字 → intent=退款 ← 单条样本的补丁
- 出现「经理/投诉」→ urgency=high ← 特例当全局规则
- 感叹号 >2 → 倾向 high ← 在拟合标注员习惯

卫生（hygiene）要进选择函数：长度比、规则范围、dev-test gap、压力集回退，和分数一起做 Pareto。

14.7× / 86%PrefPO：放任时 TextGrad 撑到 14.7 倍长、86% 在偷改任务定义
+10.0TextReg：加长度+范围正则化，反而比放任更高

13 · 防身清单六个反模式 + 三个最常见误读

反模式（摘 6/9）：

  • 没有 eval 就让模型「优化一下 prompt」→ 风格化改写
  • 只看平均分 → 掩盖 schema 漂移 / 少数类回退 / 安全退化
  • 允许 optimizer 碰 evaluator / 测试集 → reward hacking
  • 记忆无界追加 → 跨任务污染，还更贵
  • 社媒热度 / vendor 百分比当证据
  • 「仓库有测试」=「方法有效」→ 可审计 ≠ 有效

误读 top 3（社区实测高频）：

  • 「GEPA 全面替代 RL」→ 它是反思式 prompt 进化，AIME-2025 上仍输 GRPO
  • 「DSPy 是自动改提示词工具」→ 是编程框架；2023 版优化的是 demo，指令要到 MIPROv2
  • 「文字梯度 = 梯度下降」→ 是比喻：不可加、不可组合，提升常来自格式甚至 hacking

线索分流六字段：dataset / metric / baseline / model / cost / failure_cases——缺一个就降级为线索，追到一手来源再采信。

16 · 现场演示对话助手：能聊天的知识库

架构一句话：确定性内核（引导问答 + 触发规则，零幻觉）⊕ 扎根 LLM 模式（向量召回 57 条 → 受约束生成 → 强制引用洞见编号与证据等级）。

18 · 带走三句话老郭白板上最后留下的

  1. 动手之前先问「值不值得」——headroom 没超过噪声地板就别花钱，先拿零成本的笨办法垫底。
  2. 改 prompt 的本质是「证据 → 假设 → 多候选 → 验证集 → 可回滚」，不是让模型润色一遍。
  3. 更长、更多、更复杂，都不是默认的好——先证明它没带来过拟合、污染或归因困难。
