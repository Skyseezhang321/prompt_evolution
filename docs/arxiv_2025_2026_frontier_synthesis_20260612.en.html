One-Page Conclusions

  1. The era of "name a new method" (2022–2024) is over. Among the 25 papers in 2025/2026, only GEPA qualifies as "a named method that establishes a new main line." The rest produce diagnostics (is optimization worth it? did the edit actually help?), formalizations (multi-agent credit assignment), object upgrades (tool schema / pattern / codebook), self-evolution (the optimizer itself), and hygiene & regularization (anti-spurious-gain).
  2. A pre-optimization gate has become cross-paper consensus. Coin Flip gives the strongest constraint (49% of 72 optimization runs on Haiku fall below zero-shot); APO-KG gives the positive condition (the more complex the schema, the clearer the gain; cross-dataset transfer leaves only ~1% F1 gain); Prompt Repetition gives the zero-cost comparison baseline (non-reasoning mode: 47/70 significant wins, 0 losses).
  3. "Score went up ≠ actually got better" now has an actionable detection surface. PrefPO quantifies prompt hacking rate (TextGrad 86% vs PrefPO 37% vs Minimal 31%); TextReg turns overfitting into a regularization target (OOD multi-task +8~+11.8); Edit-Level causal analysis yields edit-family priors (meta_instruction × math = −0.103).
  4. The real theme of multi-agent optimization is credit assignment, not more agents. MASPO (explicit misalignment modeling), MAPRO (MAP inference + belief propagation), and Temporal/Structural Credit (block updates by role/round) converge from three independent directions.
  5. The optimizer itself is now a managed, optimized artifact. SePO ablation (removing self-improvement: 76.38→74.94), SPEAR (rigid loop significantly worse than free-form agent), and PrefPO (upgrading the optimizer +11.5 > upgrading the discriminator +6.1) support this from three directions.
  6. The real gap is reasoning models. None of the 25 papers addresses bi-level / thought-driven (the o1/R1 era treats prompts as reasoning controllers); this is consistent with the conclusions of the taxonomy completeness check and is the highest-priority registered direction for targeted search.

Shift One: Ask Whether It's Worth It Before Asking Whether the Edit Was Right (Cooling Down & Diagnostics)

This is the most distinctive line of 2025/2026: the field has begun systematically interrogating itself.

  • Coin Flip (2604) gives the strongest practical constraint: in a compound AI setup, 49% of Claude Haiku 4.5's 72 optimization runs fall below zero-shot; 14 of Nova Lite's 24 method×task means fall below zero-shot; agent-to-agent interaction shows F<1 and p>0.52 across all tests (very weak coupling — joint optimization lacks a prerequisite). Its actionable output is a pre-optimization gate: zero-shot baseline → 10–20 candidates to estimate headroom and noise floor → coupling test → stop if below threshold. The only task with positive returns across all methods is HelpSteer2 (best gain +6.8); best gains on other tasks are only +1.1 / +0.7 / +0.6.
  • PROSE (Appendix C of the same paper, note) is the authors' own controlled counter-example: structured five-stage decomposition + six-operator evolution + risk-adjusted fitness (0.70·mean score + 0.15·Sharpe + 0.15·DRO), all enabled, yet no measurable robustness advantage — "adding risk engineering on the selection side" cannot rescue fragility. Note: it is an internal baseline, not an independent publication.
  • VISTA (2603) gives reflection-based optimization a structural counter-example: on a defective GSM8K seed, GEPA drops from 23.81% to 13.50% (the root cause never enters its hypothesis space); decoupling "hypothesis generation about the root cause" from "rewriting based on the hypothesis" and running them in parallel recovers performance to 87.57%. On cross-model transfer, GEPA's optimized prompt yields only 22.74% on Qwen3-4B; VISTA still reaches 86.05%.
  • Flawed Metaphor (2512) dismantles the metaphor with destructive ablations: wrong evaluation labels typically do not reduce test performance; 100-step training shows gradient-free overfitting; high-scoring prompts may rely on prevalence hacking (strongly forbidding the minority class "unknown"). The conclusion is that method comparisons must control for the number of candidates and the selection budget; otherwise credit for "gradients" may belong entirely to "more sampling."
  • Prompt Repetition (2512) establishes the zero-cost comparison baseline: <QUERY><QUERY> in non-reasoning mode scores 47/70 significant wins and 0 losses across 7 models × 10 configurations (NameIndex: 21.33%→97.33%); padding controls have no effect; in reasoning mode only 5/28 groups win. Before any optimizer reports a gain, the first question should be whether it beats this zero-intelligence transform.
  • APO-KG (2506) is the positive-side piece of this picture: gains are clear for complex schemas / long inputs / strict formatting (ECR triple F1 up to +16%, rel=100: 0.62→0.72), but cross-dataset transfer leaves only ~1% F1 gain (in-dataset ~8% or more) — optimization is "a robustness tool under a complexity amplifier," not universal gain.

Convergence: "Is it worth optimizing?" now has an executable decision procedure (headroom / noise floor / coupling / zero-cost baseline) for the first time. "Did the edit actually help?" now has structural failure explanations (hypothesis space, prevalence hacking, transfer collapse) for the first time.

Shift Two: Expanding the Optimization Target — From a Single Prompt to an Artifact Graph

Almost none of the methodology papers in 2025/2026 ask "how to edit one instruction segment" anymore; instead they disaggregate and enlarge the optimizable target:

  • AutoPDL: the pattern itself is the most important search variable — on FEVER, Granite 13B goes from 6.5% to 74.0% (the best configuration is 3-shot ReWOO, not a better instruction); but the optimal pattern varies strongly by task and model, and GPT-4o-mini shows no MBPP+ gain on transfer.
  • JTPRO: failures in tool-use agents are usually in per-tool schema and slot descriptions, not in global instructions — on ToolACE 1000 tools: o3-mini OSR 51.27→64.46, GPT-5 62.37→73.55; ETID GPT-4o mini 46.53→66.83.
  • Modular PO: fixed schema, section-local edits only — ARC-Challenge: LLaMA-3 8B 75.0→79.10 (TextGrad only 75.67); MMLU: 57.21→61.50 (TextGrad drops to 56.40). This is the most direct experimental design for "rollback-able optimization."
  • Prompt Codebooks: optimize prompts into routable, reusable instinct units keyed by input — HotpotQA: prompt length 14.1× shorter than MIPROv2, aggregate 9.6× shorter, while performance still exceeds GEPA (IFBench 41.33, +2.72).
  • MASPO / MAPRO / Temporal-Structural Credit: three multi-agent lines jointly formalize "who to update and which round" — MASPO uses misalignment cases + joint reward (Sequential MAS average 70.39 vs baseline 65.31; removing Joint Evaluate drops to 67.77); MAPRO uses max-product belief propagation for MAP inference (removing demonstration-guided reward: HumanEval-ET 80.21→76.04, CodeContest 31.52→29.70); Temporal/Structural Credit updates only low-credit roles or rounds (MedMCQA LLaMA3-8B Debate 55.13→64.63).
  • SPEAR: even the optimizer's tools and autonomy are variables — give the optimizer a Python sandbox for confusion-matrix / groupby-style error analysis; on industrial tasks: 0.760 vs TextGrad 0.110 (job location kappa), BBH-7 average 0.938 vs GEPA 0.628; stuffing the full DataFrame into context recovers only 2% of the gap; a rigid fixed loop is significantly worse than a free-form agent; 0 of 78 archived runs fall below seed (auto-rollback protects the lower bound).

Convergence: This project's v4 report summarizes this line as "the optimization target is now an artifact graph" (prompt, pattern, tool schema, slot, role, round, codebook, optimizer tools and strategy); versioning must also be upgraded to artifact-graph level. These 8 papers are the complete primary evidence for that judgment.

Shift Three: From One-Shot Optimization to Self-Evolution and Experience Reuse

  • GEPA: the main-line anchor. Trajectory reflection + Pareto frontier; Qwen3 8B aggregate 45.23→54.85 (GRPO 48.91, MIPROv2 47.84), ~3,936 rollouts vs GRPO's 24,000; ablation shows that Pareto selection itself contributes the most (GEPA +12.44 vs SelectBestCandidate +6.05); generated prompts are 9.2× shorter than MIPROv2. Boundaries are equally clear: on AIME-2025, 32.00 falls below GRPO's 38.00; degrades to ordinary rewriting when no reliable trace is available.
  • SePO: brings the system prompt of "the prompt agent responsible for editing prompts" into the evolution loop — Manual-CoT 71.89→SePO-Generalist 76.38; removing self-improvement drops to 74.94; removing open-ended evolution drops to 72.64; multi-task pre-training cost $37.14 spread over five tasks is ~$7.43/task, and transfers to tasks not seen during pre-training (Sudoku 96.95→99.90). This is the core evidence for project insight 14, "optimizer/judge must also be versioned."
  • MemAPO: success templates and error patterns stored in separate memories (CTM/EPM dual memory) — GPT-4o-mini average 70.7% vs TextGrad 63.6%, at 58.6% lower cost (average $0.31 vs $0.70); AQuA-RAT with dual memory: 61.7→82.5 (CTM-only / EPM-only reach only 80.5 / 77.9).
  • DistillPrompt: examples are first distilled into task principles, then compressed and aggregated; average improvement across all datasets relative to Grips: +20.12% — another low-cost path for experience reuse.
  • Scaling Textual Gradients: full-batch textual gradients hit the context wall at ~50 examples; using historically high-scoring prompts as a momentum distribution for sampling (TSGD-M) — TREC 81.92→83.36, GSM8K 93.15→94.04 — "historical experience goes into the sampling distribution, not into the context."

Convergence: The engineering implication of self-evolution is not "let the system modify itself freely" but rather bringing all artifacts on the optimizer side (its prompts, tools, memory, sampling distribution) under the same versioning, evaluation, and rollback discipline applied to task prompts. SePO's ablation, SPEAR's rigid-loop comparison, and PrefPO's finding that "upgrading the optimizer (+11.5) is more effective than upgrading the discriminator (+6.1)" support the same conclusion from three independent directions.

Shift Four: From "Score Went Up" to "Actually Got Better" (Hygiene, Regularization & Constraints)

  • PrefPO: hygiene is built into evaluation — BBH average 0.875 (MIPROv2 0.873, GEPA 0.870, TextGrad 0.850), while prompt length inflates only 4.7× (TextGrad 14.7×, Minimal variant 2.2×), repetition rate 0.044 (TextGrad 0.117, Minimal 0.012), prompt hacking rate 37% (TextGrad 86%, Minimal 31%, unoptimized baseline 12.8%). A prompt that is long, repetitive, and games the metric does not count as a successful optimization.
  • TextReg: makes overfitting a regularization target — generates a regularization gradient alongside the task gradient to control length growth and rule narrowing; on OOD relative to TextGrad, multi-task +8~+11.8 (Phi-3.5-Mini Logical Deduction 5obj +11.8), while TextGrad/REVOLVE frequently fall below unoptimized CoT on OOD. Scope limitation is explicit: single-turn, rule-clear reasoning tasks only.
  • Edit-Level causal analysis: observational analysis of 2,095 DSPy pairs + 17,708 TextGrad/GEPA pairs — meta_instruction × math = −0.103, clarity_constraint × logical = −0.083, extraneous_load × sequential = −0.060; 83% of association directions are stable under leave-one-out. Different tasks respond to the same edit family in opposite directions; "the clearer and more complex, the better" is a misconception.

Convergence: Combined with the prevalence hacking cases from Flawed Metaphor, these four papers together give a directly deployable "spurious-gain detection surface": length ratio, repetition ratio, seed similarity, edit family, minority-class performance, OOD/stress delta, hacking flag — these are the origin of this project's v4 engineering layer's release gate and ledger fields.

Survey Layer: The Field's Self-Positioning

The three surveys (all 2025) produce no new methods but accomplish two things:

  1. Unified frameworks: APO Survey uses a 5-stage process anatomy (seed → evaluation feedback → candidate generation → selection and retention → iteration depth); APE Survey uses three axes (optimization space × objective function (including constraints) × optimization method); Context Engineering Survey positions prompt optimization as a fundamental component under "Context Retrieval & Generation" in the larger context engineering picture — confirming that this project's scope of "black-box, discrete, natural-language prompt" is a reasonable narrow slice.
  2. Jointly naming the frontier: constrained optimization (formalizing length/readability constraints — naturally adjacent to the hygiene empirics of Shift Four); multi-task and negative transfer; online/task-agnostic optimization; multi-objective Pareto. Only APE Survey names bi-level / thought-driven (o1/R1 reasoning models as high-level controllers) — and this is precisely the direction absent from all 25 empirical papers.

All three have been used in the taxonomy external completeness check: 7 clusters show no missing wholesale sections; what is missing is the above-mentioned frontier.

Two Tensions: How Inconsistencies Converge into Applicability Conditions

The most valuable thing in 2025/2026 is not individual paper conclusions but the fact that "surface contradictions" between papers have converged into applicability conditions:

  1. Reflection is useful (GEPA) vs. reflection is harmful (VISTA). Unified condition: whether the feedback signal contains trajectory-level diagnostic information and whether the true root cause falls within the optimizer's hypothesis space. Trace available, root cause visible → reflection-based optimization is extremely sample-efficient; root cause structurally absent → more reflection means more drift, requiring decoupling of hypothesis generation from rewriting.
  2. Large gains (JTPRO +13.19pp / AutoPDL +67.5pp / SPEAR 0.110→0.760) vs. coin flip (Coin Flip: 49% below zero-shot). Unified condition: task structure. Complex schema, tool selection, strict formatting, latent capability not activated by default → optimization has exploitable structure. Free text, zero-shot already near ceiling, dev set small and noisy → optimization is expensive search through noise.

These two "tension → condition" convergences are something you can only obtain by reading the 2025/2026 literature as a whole. They are also the literature foundation for this project's six selection questions and pre-optimization gate.

What 2025/2026 Has Not Yet Covered (Honest Inventory)

  • bi-level / thought-driven: not a single paper among the 25; only APE Survey names it conceptually. The optimization of "prompt as reasoning controller" in the reasoning-model era is a wholesale-absent direction in the current library, registered as the highest-priority targeted search (top-80 sampling may simply not surface these — different search terms are needed).
  • Task-agnostic / online optimization: inference-time optimization without a dev set in production, only tangentially touched by memory-style work.
  • Theoretical formulation for constrained optimization: there is substantial empirical work on hygiene/length (Shift Four), but no representative paper with a formal solution in the style of "Γ(P)≤κ" has entered the library.
  • Multi-task negative transfer: cross-task results from MemAPO/SePO are positive cases; systematic negative-transfer research is not covered.
