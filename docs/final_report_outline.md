# 最终报告结构

更新时间：2026-06-08

## 报告目标

最终报告要回答两个问题：

1. 截至 2026-06-08，prompt 优化与 prompt 自进化领域的前沿状态是什么。
2. 在工程实践中，哪些方案或建议值得实际执行，前提、成本、风险和验证方式分别是什么。

报告不追求穷尽所有论文，也不声称初步实验能证明长期稳定收益。报告的价值来自可追溯证据、清晰判断、可执行方案和诚实的风险边界。

## 核心问题

- 当前前沿方法如何分类：自动 instruction 搜索、文本梯度、反思式进化、prompt-as-program、记忆型自进化、agent/context 优化。
- 行业实践正在收敛到什么：eval-driven development、prompt versioning、observability、人工审核、rollback、context engineering。
- 哪些结论已有较强证据，哪些只是初步趋势。
- 哪些方法适合作为短期落地方案，哪些更适合作为中长期研究方向。
- 初步实验能验证哪些关键判断，不能验证哪些判断。

## 建议结构

1. 执行摘要：用一页说明主要结论、推荐方案和风险。
2. 研究范围与方法：说明来源筛选、证据等级、实验边界和不覆盖内容。
3. 前沿状态地图：按方法类别总结学术论文和工程框架。
4. 行业经验总结：总结官方文档、工具实践和生产治理经验。
5. 关键判断：逐条列出判断、证据来源、反例或不确定性。
6. 可执行方案：给出 2-3 个方案，每个方案包含适用场景、实施步骤、所需数据、评估指标、成本和回滚策略。
7. 初步实验：描述实验目标、数据、prompt 版本、模型、指标、结果、失败案例和限制。
8. 风险与治理：覆盖 eval 过拟合、judge 偏差、prompt 漂移、安全退化、成本失控和跨模型脆弱性。
9. 后续路线：说明如果继续投入，应优先补哪些数据、实验和工程能力。

## 证据等级

- A：原始论文、官方文档或开源项目文档，并已完成结构化笔记。
- B：多个独立来源重复出现的行业实践或工程经验，已记录来源和适用条件。
- C：本项目初步实验观察，有运行记录、样例输出和失败案例。
- D：基于资料综合形成的推测，尚未有充分证据，必须标注为待验证。

最终报告中的每个关键判断都必须标注证据等级。不能把 D 级推测写成确定结论。

## 可执行方案字段

每个方案至少写清：

```yaml
name:
problem:

not_recommended_when:

implementation_steps:
evaluation_metrics:
expected_benefit:
cost_and_latency:
risks:
rollback_plan:

next_experiment:
```

## 验收标准

最终报告完成时应满足：

- 能说明当前前沿的主要方向和代表方法。
- 能把论文和行业实践转化为可执行建议，而不是只做摘要。
- 至少包含 2 个可执行方案或建议。
- 至少包含 1 个初步实验或可复现实验设计。
- 每个关键结论都能追溯到来源、实验或明确标注为待验证。
- 明确列出不确定性、反例和后续验证路径。
